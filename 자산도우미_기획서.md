# 자산도우미 기획서

> 부제: "내 소비가 키우는 또 하나의 나"
> 기준 문서: [자산도우미_아키텍처_옵션B.md](자산도우미_아키텍처_옵션B.md)의 권장안을 따른다. 초기 구현 기준은 Fly.io, Neon, APScheduler, Redis, Sentry다.

---

## 1. 프로젝트명
- **자산도우미** — 부제: "내 소비가 키우는 또 하나의 나"

## 2. 문제 정의
- 모바일 결제·후불결제 확산으로 **소비 행동과 잔액 변화의 인지 연결고리가 약화**됨
- 가계부 앱은 *기록 중심*이라 결제 직전 개입이 어려움
- 챌린지·만보기형 앱은 외재적 보상에 머물러, *내재적 동기로의 전환을 설계가 받쳐주지 못함*
- 결과적으로 사회초년생은 비상금·정기 저축 같은 **작은 자기 통제 습관**을 만들 계기가 부족

## 3. 타깃
- **1차 (단일 집중)**: 사회초년생 만 23~29세
  - 월 가용 소득 50~250만 원
  - 비상금/저축 목표는 있으나 추적이 산발적
  - 후불결제·구독 누수에 노출
- **후속 확장 (현 단계 미포함)**: 대학생, 청소년 모드 — 페르소나·리스크 구조가 달라 별도 라인. 청소년 모드는 **임상 자문 전 도입하지 않음**

## 4. 핵심 가치 제안
**"내 소비·저축 행동이 가상의 분신을 키우거나 시들게 한다 — 단, 사용자가 강도를 정한다."**

- 실제 자산을 1:1 미러링하지 않고, 행동 결과를 추상화한 별도 지표(분신 HP·성장도)로 가시화
- 결제 직전·직후 **"한 박자 멈춤"** 모먼트 형성
- 경쟁이 아니라 **자기 통제 + 선택적 동행**
- **정신건강 가드레일**
  - 시들기 강도/표현톤(부드러움 ↔ 자극적), 비교 표시 여부, 알림 빈도를 사용자가 직접 조절
  - 우울/강박 성향 자가체크 옵션 제공
  - "실패", "당신은 X" 같은 비난적 카피 금지 정책

## 5. 핵심 기능

### 5-1. 데이터 수집 — 단계적 + 한계 명시

| 단계 | 방식 | 신선도 |
|---|---|---|
| MVP | 수동 입력 + 모의 데이터 | 즉시 |
| Beta | 오픈뱅킹 잔액·이체 + (선택) 카드사 즉시결제 알림 동의 | 분 단위 |
| 정식 | 마이데이터 사업자 위탁 + 카드사 즉시알림 | 표준 API는 일 단위, 즉시알림은 분 단위 |

- "실시간"이라 표현하지 않고 **"준실시간(분 단위)"** 으로 정직하게 표기
- SMS 파싱 의존 ❌ (OS 정책상 비현실) — 카드사 푸시 동의·즉시결제 알림 API 사용

### 5-2. 분신 시스템
- 분신 종류: 식물 / 캐릭터 / 미니 도시 / **텍스트 모드**(메타포 거부 사용자용)
- 두 축
  - **HP**: 최근 7일 소비를 *사용자 본인의 평소 패턴*과 비교한 충동성 지수 (절대 금액 ❌, 자기 베이스라인 비교)
  - **성장도**: 누적 저축·자동이체·학습 카드 완료의 일관성
- 회복 액션: 저축 실행, 자동이체 룰 추가, 1~2분 학습 카드 완료, 소비 일기
- **가드레일**
  - HP 0이어도 "실패" 표현 금지, 회복 액션 항상 노출
  - 시들기 시각 효과는 **강도 3단계** 사용자 선택
  - 시들기·HP 알림 빈도 **일 N회 상한** 사용자 설정

### 5-3. 멈춤 카드 — 사전약속 기반, 기술 한계 인정

앱이 결제 시점을 직접 가로채지 못한다는 점을 인정하고, 다음 3가지 트리거로 구성:

1. **사전 임계 알림**: 사용자가 정한 *일/주 누적금액·카테고리·시간대* 도달 시 푸시
2. **결제 시뮬레이터**: 큰 결제 전, 앱 내에서 "이거 살까?" 시뮬레이션 → 분신 영향 미리보기
3. **카드사 즉시 알림 후 N초 회수창**: 결제 알림이 도착한 직후 짧은 시간 내 "계획된 소비였나요?" 회고 노출 (가능한 카드사 한정)

- 결제 직후 자동 차단 같은 *과장된 약속은 하지 않음*
- 임계가 너무 느슨해지지 않도록 매 주 *사용자 패턴 기반 자동 추천* 제공

### 5-4. 동행 기능 — 랭킹 없이, 사용자 주도
- **친구 소그룹(2~6명) 챌린지**: 동의자만 입장, 우열 표시 ❌, 응원 위주
- **봇 메이트**: 친구 모집이 부담스러운 사용자용
- **과거 자아 비교**: *옵션*, 기본 비활성. 켜더라도 일/주 단위만, 매일 강제 노출 ❌
- **경제력 비대칭 보호**: 챌린지 목표는 *금액 절대치* 대신 *본인 평소 대비 % 변화* 권장

### 5-5. 학습·자동저축 룰 빌더 — 가역성 우선
- 1~2분 카드형 학습(비상금 3원칙·복리·ETF 입문 등) → 분신 성장도 +1
- 자동저축 룰 빌더 (오픈뱅킹 출금이체 동의 기반)
  - **가역성 보장**: 룰 일시정지(1탭), 다음 실행 시뮬레이션 미리보기, 월세·카드대금 일정과 충돌 경고
  - 잔액이 사용자 지정 하한 이하일 때 자동 일시정지
  - 첫 실행 전 **24시간 유예**(취소 가능)

## 6. 기대효과

효과 크기 단정 대신 **행동 단계의 변화**로 표현:

- **즉시(1~4주)**: 사용자 본인이 정의한 임계에 대해 멈춤 카드 노출 → 결제 결정 *이전*에 한 번 더 의식적 점검 단계 생김
- **단기(1~3개월)**: 자동저축 룰 1개 이상 가동 + 일시정지/충돌 경고 학습 → 자동화의 *통제권*이 사용자에게 있다는 효능감 형성
- **중기(3~6개월)**: 그룹 챌린지·과거 자아 비교·학습 카드 누적이 분신 성장으로 연결되며, **외재적 동기에서 내재적 동기로의 전환을 설계가 의도적으로 유도** — 단, 전환은 *가설*이며 사용자 인터뷰·자기보고로 검증 대상
- **장기(6개월+)**: 마이데이터 통합 시 카테고리·정기지출 분석으로 개인화 코치 가능

**부정적 결과 가능성도 인정**: 일부 사용자는 분신 메타포에 거부감 또는 정신적 부담을 느낄 수 있어, 언제든 **메타포 OFF + 텍스트 모드**로 전환할 수 있는 **이탈 경로**를 항상 보장.
